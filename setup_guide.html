WeatherCore — Setup Guide

How to Run
the App for Real

A complete step-by-step guide from a fresh machine to a live running dashboard — pick your operating system below.

Phase A
A
Install System Dependencies

Install Python, MySQL and Git if not already present.

01
Update system & install Python 3.11+
apt update → install python3, pip, venv
▾
bash
sudo apt update && sudo apt upgrade -y sudo apt install python3 python3-pip python3-venv git -y # Verify python3 --version # should say Python 3.11+ or 3.12+
02
Install MySQL 8.0
Install MySQL server and run the security wizard
▾
bash
sudo apt install mysql-server -y sudo systemctl start mysql sudo systemctl enable mysql # auto-start on reboot # Run the security wizard — set a root password when prompted sudo mysql_secure_installation
ℹ
Security wizard prompts Answer: MEDIUM password policy → Yes → Yes → Yes → Yes. This removes test databases and anonymous users.
Phase B
B
Create the Database & User

Create a dedicated MySQL user and database. Never use root for the app.

03
Log into MySQL as root
sudo mysql (Ubuntu doesn't need a root password here)
▾
bash
sudo mysql

You should see the mysql> prompt. Now run the SQL commands below inside it.

04
Create database, user & grant permissions
Run inside the mysql> prompt
▾
sql — paste inside mysql> prompt
CREATE DATABASE weathercore CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci; CREATE USER 'weatheruser'@'localhost' IDENTIFIED BY 'StrongPass!2025'; GRANT ALL PRIVILEGES ON weathercore.* TO 'weatheruser'@'localhost'; FLUSH PRIVILEGES; EXIT;
⚠
Change the password Replace StrongPass!2025 with your own password. Use the same password in your .env file in the next phase.
05
Run the schema file
Creates all 6 tables, views, stored procedure and seed data
▾

Navigate to the folder where you saved schema.sql and run:

bash
mysql -u weatheruser -p weathercore < schema.sql # Enter your password when prompted # You should see output like: # status | stations_seeded | config_keys # ✓ Schema created: 8 tables/views | 8 | 15
Phase C
C
Set Up the Python Project

Create a virtual environment and install all Python packages.

06
Create project folder structure
One folder, two subfolders: backend files + frontend HTML
▾
bash
mkdir -p weathercore/frontend cd weathercore # Move your downloaded files into place: # schema.sql → weathercore/schema.sql # backend.py → weathercore/backend.py # app.py → weathercore/app.py # requirements.txt→ weathercore/requirements.txt # index.html → weathercore/frontend/index.html
07
Create virtual environment & install packages
Isolated Python environment — keeps your system Python clean
▾
bash — run from inside the weathercore/ folder
python3 -m venv venv source venv/bin/activate # Your prompt now shows (venv) — this is correct pip install --upgrade pip pip install -r requirements.txt # This installs: flask, flask-cors, mysql-connector-python, # apscheduler, requests, python-dotenv, pandas # Takes about 1-2 minutes
08
Create your .env file
Copy the example and fill in your database password
▾
bash
cp env_example.txt .env nano .env # or use any text editor

The only line you must change is DB_PASS. Set it to the password you used in Step 04. Everything else is optional.

.env — minimum required changes
DB_HOST=localhost DB_USER=weatheruser DB_PASS=StrongPass!2025 # ← change this to your actual password DB_NAME=weathercore # Optional — leave blank to skip OWM (Open-Meteo still works) OWM_API_KEY= NOAA_TOKEN= FLASK_ENV=development PORT=5000
Phase D
D
Run the App

Start the Flask server — it also starts the background scheduler automatically.

09
Start Flask
One command — serves API + dashboard + starts data polling
▾
bash — from inside weathercore/ with venv active
python app.py

You should see output like this — if you do, it's working:

expected terminal output
✓ MySQL connection pool created (size=5) Alert engine loaded 5 threshold rules ✓ Scheduler started → Ingestion: every 15 minutes → Summaries: daily at 00:05 UTC → Thresholds: reload every 60 minutes ═══ Starting ingestion cycle ═══ Active stations: 8 ✓ KE-NBO-001 open_meteo → 21.4°C obs_id=1 ✓ KE-MBA-001 open_meteo → 30.2°C obs_id=2 ✓ Running on http://0.0.0.0:5000
10
Open the dashboard
Visit in your browser
▾
browser
http://localhost:5000
✓
You should see The dark WeatherCore dashboard loading with real station data. The map shows markers for all 8 stations. KPI cards show live temperature, humidity, wind and UV readings. Charts populate as soon as the first 15-minute fetch completes.

Troubleshooting
?
Common Errors & Fixes
ErrorCauseFix
Access denied for user 'weatheruser' Wrong password in .env Open .env, make sure DB_PASS matches exactly what you set in Step 04
Unknown database 'weathercore' Schema not imported yet Re-run mysql -u weatheruser -p weathercore < schema.sql
ModuleNotFoundError: No module named 'flask' Virtual env not activated Run source venv/bin/activate (Linux/Mac) or venv\Scripts\activate (Windows) before python app.py
Address already in use — port 5000 Another process on port 5000 Run export PORT=5001 then python app.py, visit localhost:5001
Dashboard shows "Offline" red dot Flask is not running Switch back to the terminal and check for error messages. Re-run python app.py
Charts are empty after 15 minutes API fetch failed Check terminal for open_meteo failed lines. Verify internet connection. Open-Meteo requires no key so should always work.
mysql: command not found (Windows) MySQL not in PATH Add C:\Program Files\MySQL\MySQL Server 8.0\bin to your Windows PATH environment variable
